🏛️ Pantheon Dashboard

O painel de controle visual do TRIAD — gerenciando personas, versionando system prompts e sincronizando com o Hermes em tempo real.

O que é

Pantheon é uma interface web local (roda em localhost) que centraliza o gerenciamento de todas as personas do seu ecossistema TRIAD. Em vez de editar arquivos .md manualmente espalhados em ~/.hermes/skills/, você tem um painel visual com grid de personas, editor integrado, histórico de versões e botão de sincronização com o Hermes ativo.

Por que aprender

Quando você tem 5 ou mais personas ativas, gerenciar via terminal fica impraticável. Pantheon resolve isso com visibilidade — você vê todas as personas, quais estão ativas, quando foram modificadas e qual versão está em uso.

Conceitos-chave

  • •Interface web local em localhost
  • •Grid visual de personas ativas
  • •Editor integrado de system prompts
  • •Sincronização com ~/.hermes/skills/

O que é

O Pantheon é organizado em quatro painéis principais: (1) Grid de Personas — cards com nome, descrição, status e data de modificação; (2) Editor de Prompts — editor Markdown com syntax highlighting; (3) Histórico de Versões — timeline de mudanças por persona com diff visual; (4) Painel de Sincronização — status de quais personas estão carregadas no Hermes ativo.

Por que aprender

Cada painel resolve um problema específico do ciclo de vida de personas. Entender a estrutura permite navegar com eficiência e saber onde ir para cada tipo de tarefa — editar, revisar histórico ou sincronizar.

Conceitos-chave

  • •Grid de Personas com cards informativos
  • •Editor Markdown com syntax highlighting
  • •Histórico de versões com diff visual
  • •Status de sincronização em tempo real

O que é

O fluxo para adicionar uma nova persona no Pantheon: (1) clique em "Nova Persona", (2) defina nome e descrição curta, (3) escreva o system prompt no editor (usando template sugerido), (4) adicione metadados — modelo preferido, temperatura, casos de uso, (5) clique em "Testar" para validar com uma pergunta de exemplo, (6) "Publicar" sincroniza para ~/.hermes/skills/ e ativa no Hermes.

Por que aprender

O fluxo guiado do Pantheon reduz o tempo de criar uma nova persona de 20+ minutos (editar arquivos manualmente) para menos de 5 minutos, com garantia de que o formato está correto e a persona está ativa.

Conceitos-chave

  • •Template de system prompt integrado
  • •Metadados de persona (modelo, temperatura)
  • •Teste antes de publicar
  • •Deploy com um clique em Publicar

O que é

Cada alteração salva em uma persona cria automaticamente uma versão numerada com timestamp e comentário opcional. O painel de Histórico mostra todas as versões em ordem cronológica, permite comparar duas versões (diff visual) e fazer rollback para qualquer versão anterior com um clique — sem perder o histórico.

Por que aprender

System prompts evoluem — e às vezes uma mudança quebra o comportamento esperado. Versionamento com rollback permite experimentar com segurança: se a nova versão piorou os resultados, você volta à anterior em segundos.

Conceitos-chave

  • •Versão numerada com timestamp automático
  • •Diff visual entre versões
  • •Rollback sem perda de histórico
  • •Comentário de versão para rastreabilidade

O que é

Quando você publica uma persona ou atualiza uma existente no Pantheon, o sistema envia um sinal para o Hermes Agent em execução para recarregar aquela skill específica. Isso permite hot reload — a persona atualizada fica disponível na próxima invocação sem precisar reiniciar o Hermes ou reabrir o terminal.

Por que aprender

Sem sincronização em tempo real, editar uma persona requereria reiniciar o Hermes — interrompendo tarefas em andamento. O hot reload permite iterar no prompt enquanto o agente continua operando.

Conceitos-chave

  • •Hot reload de skills individuais
  • •Sinal de sincronização para Hermes ativo
  • •Sem interrupção de tarefas em andamento
  • •Status de sincronização visível no painel

O que é

Pantheon exporta skills no formato compatível com o diretório userSettings do Claude Code (~/.claude/skills/). Quando você edita uma persona no Pantheon e ativa a integração com Claude Code, a skill fica disponível tanto no Hermes quanto no Claude Code com o mesmo prompt — mantendo consistência sem duplicação manual.

Por que aprender

Manter personas em dois lugares (Hermes e Claude Code) manualmente leva a divergência. Pantheon como fonte única de verdade garante que a persona Orpheus, por exemplo, tem exatamente o mesmo system prompt nos dois contextos.

Conceitos-chave

  • •Exportação para ~/.claude/skills/
  • •Fonte única de verdade para system prompts
  • •Sem divergência entre Hermes e Claude Code
  • •Sincronização bidirecional configurável

O que é

Pantheon suporta customizações de layout e fluxo: reordenar painéis, filtrar personas por tag ou projeto, criar grupos de personas relacionadas, configurar atalhos de teclado para ações frequentes e exportar configurações para compartilhar com outros membros da equipe que usam o mesmo sistema TRIAD.

Por que aprender

À medida que o ecossistema cresce (15, 20+ personas), organização é essencial. Agrupamento por projeto e tags reduz o tempo de encontrar a persona certa de minutos para segundos.

Conceitos-chave

  • •Tags e grupos de personas
  • •Filtros por projeto ou categoria
  • •Atalhos de teclado configuráveis
  • •Export de configurações para equipe
