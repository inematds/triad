🔗 Integrações Externas

Conectando o Hermes ao mundo — Telegram como interface mobile, N8N para orquestração e Zapier para integrações sem código.

O que é

O Hermes sozinho é poderoso, mas seu impacto multiplica quando conectado ao restante do seu stack: Telegram para acionar tarefas do celular, N8N para orquestrar fluxos complexos com múltiplos sistemas, e Zapier para conectar centenas de ferramentas SaaS sem código. A integração transforma Hermes de ferramenta local em hub central de automação.

Por que aprender

Sem integrações, o Hermes funciona apenas quando você está no terminal. Com integrações, qualquer evento em qualquer sistema pode disparar uma tarefa — o agente passa a trabalhar no contexto de todo o seu ecossistema digital.

Conceitos-chave

  • •Hermes como receptor de webhooks
  • •Integrações como canais de entrada
  • •Resposta via canal de origem ou arquivo
  • •Stack de automação conectado

O que é

Para criar um bot Telegram conectado ao Hermes: (1) abra o BotFather no Telegram e envie /newbot, (2) defina nome e username do bot, (3) receba o token de acesso, (4) adicione TELEGRAM_BOT_TOKEN no arquivo .env do Hermes, (5) execute hermes integrations telegram enable para ativar o listener, (6) envie /start para o bot no Telegram para confirmar conexão.

Por que aprender

Telegram é o canal de acesso mobile mais prático para o Hermes. Com o bot configurado, você delega tarefas de qualquer lugar — no celular, sem abrir terminal, enquanto está em trânsito ou numa reunião.

Conceitos-chave

  • •BotFather para criação do bot
  • •TELEGRAM_BOT_TOKEN no .env
  • •Listener ativado via hermes integrations
  • •Resposta assíncrona no mesmo chat

O que é

Quando você envia uma mensagem ao bot Telegram, ela chega ao Hermes como um webhook POST. O Hermes identifica o chat_id do remetente (verificação de segurança), extrai o texto da mensagem, converte em instrução de job, executa, e responde de volta ao mesmo chat com o resultado — o ciclo completo fica transparente no chat.

Por que aprender

Entender o fluxo técnico permite depurar quando algo falha e personalizar o comportamento — como adicionar comandos especiais (/resumo, /status, /jobs) que o bot reconhece como atalhos para ações frequentes.

Conceitos-chave

  • •Webhook POST do Telegram
  • •Verificação de chat_id autorizado
  • •Comandos especiais como atalhos
  • •Resposta assíncrona no chat de origem

O que é

N8N é uma plataforma de automação open-source com interface visual de fluxos (nodes e edges). Diferente do Zapier (cloud-only, pago por tarefa), o N8N pode rodar localmente, é self-hosted e tem custo fixo. Com N8N você cria pipelines visuais que conectam qualquer serviço — e pode incluir o Hermes como um node via HTTP Request para o webhook.

Por que aprender

N8N resolve automações complexas que envolvem múltiplos sistemas e lógica condicional — coisa que um webhook simples não faz. É o orquestrador ideal para fluxos como "novo lead → enriquece dados → Hermes escreve email personalizado → envia via SendGrid".

Conceitos-chave

  • •N8N self-hosted vs cloud
  • •Fluxos visuais com nodes e edges
  • •Hermes como node via HTTP Request
  • •Lógica condicional e ramificações

O que é

No N8N, adicione um node "HTTP Request" com método POST, URL apontando para o endpoint do Hermes (http://localhost:PORT/webhook), header Authorization com seu token e body JSON contendo task (instrução), skill (persona a usar) e callback_url (onde enviar o resultado). O Hermes processa e chama de volta o callback com o output.

Por que aprender

A configuração do node N8N é o único passo técnico necessário. Depois disso, qualquer fluxo que o N8N processa pode incluir o Hermes como etapa de inteligência — sem código adicional.

Conceitos-chave

  • •Endpoint http://localhost:PORT/webhook
  • •Header Authorization com token
  • •Body JSON com task, skill, callback_url
  • •Resposta assíncrona via callback

O que é

Zapier é a alternativa cloud-managed ao N8N — mais simples de configurar (sem instalação), com mais de 5000 integrações nativas e plano gratuito para fluxos básicos. A limitação é custo variável por tarefa e execução em cloud da Zapier (sem controle total dos dados). Para o Hermes, usa o mesmo mecanismo: Zap com ação Webhook POST para o endpoint do Hermes.

Por que aprender

Zapier é a escolha certa para equipes não-técnicas que precisam de integrações rápidas sem instalar N8N. O mesmo padrão de webhook funciona — a diferença é operacional, não técnica.

Conceitos-chave

  • •Zapier para equipes não-técnicas
  • •5000+ integrações nativas
  • •Custo variável vs N8N self-hosted fixo
  • •Mesmo padrão webhook funciona

O que é

O pipeline completo funciona assim: você envia "Analisa os dados de vendas desta semana e recomenda 3 ações" pelo Telegram → Hermes recebe via webhook, carrega soul.md + skill de análise → aciona padrão Triad (DeepSeek rascunha, Claude valida, GPT sintetiza) → Hermes consolida e envia a análise de volta para o Telegram → você lê no celular 5 minutos depois.

Por que aprender

Ver o pipeline completo em operação é o momento em que tudo se conecta. Entender cada handoff — do Telegram ao Hermes ao Triad e de volta — permite otimizar a latência total e depurar exatamente onde uma falha ocorreu.

Conceitos-chave

  • •Telegram como interface de entrada
  • •Hermes orquestrando Triad
  • •Resposta consolidada de volta ao canal
  • •Latência total do pipeline end-to-end
