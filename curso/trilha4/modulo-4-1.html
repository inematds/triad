🏗️ Arquitetura do Sistema

Como os componentes do TRIAD se conectam — Hermes, OpenRouter, soul.md, Pantheon e Gemini CLI formando uma máquina coesa.

O que é

O sistema TRIAD tem cinco componentes fundamentais: Hermes Agent (orquestrador central), o padrão Triad (três modelos para decisões robustas), OpenRouter (gateway multi-modelo via BYOK), soul.md (memória persistente do contexto pessoal) e Pantheon (dashboard visual para gerenciar personas e prompts). Cada componente tem papel específico e se integra com os demais.

Por que aprender

Entender a arquitetura completa permite diagnosticar problemas, otimizar cada camada e escalar o sistema com consciência. Sem o mapa mental dos componentes, você usa o sistema como caixa-preta e perde capacidade de customização.

Conceitos-chave

  • •Hermes como orquestrador central
  • •Triad como workflow de três modelos
  • •OpenRouter como hub de roteamento
  • •soul.md como memória persistente

O que é

OpenRouter é um gateway que unifica o acesso a dezenas de modelos de IA (Claude, GPT, DeepSeek, Gemini, Llama) via uma única API. Com o modelo BYOK (Bring Your Own Keys), você conecta suas próprias chaves de cada provider e paga diretamente — sem markup da plataforma intermediária. Uma configuração, todos os modelos.

Por que aprender

OpenRouter é o que torna o padrão Triad operacional. Sem ele, você precisaria gerenciar APIs separadas para cada modelo. Com ele, trocar de Claude para DeepSeek para GPT é uma mudança de uma linha no código.

Conceitos-chave

  • •API unificada para múltiplos providers
  • •BYOK — sem markup intermediário
  • •Roteamento dinâmico por modelo
  • •Configuração via variável OPENROUTER_API_KEY

O que é

soul.md é um arquivo Markdown que vive em ~/.hermes/soul.md e funciona como o "estado persistente" do Hermes. Contém identidade profissional do usuário, metas, ritmo de trabalho, projetos ativos, personas preferidas, decisões tomadas e lições aprendidas. É carregado automaticamente como contexto em cada sessão.

Por que aprender

soul.md é o diferencial que transforma Hermes de chatbot em colaborador. Sem ele, cada sessão começa do zero. Com ele, o agente sempre conhece quem você é, o que importa e como você decide.

Conceitos-chave

  • •Arquivo em ~/.hermes/soul.md
  • •Carregamento automático em cada sessão
  • •Contexto pessoal estruturado em seções
  • •Atualização incremental conforme o agente aprende

O que é

Pantheon é o painel visual do TRIAD — uma interface web local onde você gerencia todas as personas (skills), versiona system prompts, testa configurações e sincroniza com o Hermes. É o ponto de controle central para quem opera múltiplas personas e precisa de visibilidade sobre o que cada agente faz.

Por que aprender

Sem Pantheon, gerenciar múltiplas personas é confuso — arquivos espalhados, sem histórico de versões, sem visão consolidada. Com Pantheon, você tem controle visual total sobre o ecossistema de agentes.

Conceitos-chave

  • •Interface web local para gestão de personas
  • •Versionamento de system prompts
  • •Sincronização com ~/.hermes/skills/
  • •Dashboard de atividade de personas

O que é

Gemini CLI é uma ferramenta de linha de comando que permite ao Hermes processar conteúdo multimodal — imagens, screenshots, vídeos curtos e documentos PDF — localmente, sem upload para a nuvem. Opera com tier gratuito generoso do Google AI Studio e se integra como ferramenta bash dentro do Hermes.

Por que aprender

A maioria dos agentes de texto não consegue analisar imagens ou vídeos. Gemini CLI preenche essa lacuna, permitindo que Hermes lide com conteúdo visual — analisar dashboards, capturar screenshots de sites, transcrever vídeos de reunião.

Conceitos-chave

  • •Processamento local de imagens e vídeos
  • •Tier gratuito do Google AI Studio
  • •Integração via ferramenta bash no Hermes
  • •Complemento multimodal ao stack de texto

O que é

Uma tarefa no TRIAD segue o fluxo: (1) usuário envia instrução ao Hermes, (2) Hermes carrega soul.md + skill relevante, (3) aciona modelo via OpenRouter com context adequado, (4) se tarefa complexa, executa padrão Triad com dois ou três modelos, (5) consolida resultado, (6) entrega e atualiza logs em ~/.hermes/jobs/.

Por que aprender

Mapear o fluxo completo revela onde cada componente entra e onde podem ocorrer gargalos ou falhas. Com esse mapa, você otimiza inputs, escolhe o modelo certo para cada etapa e depura problemas com precisão.

Conceitos-chave

  • •Pipeline input → contexto → modelo → resultado
  • •Carregamento condicional de skills
  • •Logs em ~/.hermes/jobs/
  • •Decisão de acionar padrão Triad

O que é

A arquitetura TRIAD tem pontos de extensão deliberados: (1) novas skills via ~/.hermes/skills/, (2) novos modelos via OpenRouter sem mudar código, (3) webhooks de entrada via N8N ou Zapier, (4) ferramentas bash customizadas registradas no CLAUDE.md, e (5) novas seções no soul.md para novos tipos de contexto.

Por que aprender

Conhecer os pontos de extensão corretos evita gambiarras. Em vez de modificar o núcleo do sistema, você expande de forma limpa e reversível — adicionando capacidades sem quebrar o que já funciona.

Conceitos-chave

  • •Skills como unidades de extensão
  • •Ferramentas bash customizadas
  • •Webhooks como entradas externas
  • •Modelos substituíveis via configuração

O que é

O TRIAD é projetado para custo mínimo: DeepSeek V4 processa tarefas pesadas overnight por cerca de 1% do custo de Claude Opus. Gemini CLI usa tier gratuito para multimodal. Claude Opus é reservado para decisões críticas. Uma operação de volume médio (50 tarefas/mês) custa menos de $10 com esse stack otimizado.

Por que aprender

Planejar custos antecipadamente evita surpresas. Entender a relação custo-valor de cada modelo permite alocar o orçamento de IA de forma inteligente — usando modelos caros só quando o ROI justifica.

Conceitos-chave

  • •DeepSeek V4 para volume overnight
  • •Claude Opus para decisões críticas
  • •Gemini gratuito para multimodal
  • •Alocação de orçamento por tipo de tarefa
