📚 O que é uma skill no Hermes e como funciona

Skills são configurações de comportamento especializado — combinando system prompt, modelo específico, fluxo de execução e ferramentas disponíveis. Quando você ativa uma skill, o Hermes muda seu comportamento inteiro para seguir o padrão definido nela.

O que é

Uma skill é um módulo de comportamento especializado no Hermes — composto por system prompt, modelo específico, ferramentas disponíveis e fluxo de execução. Ativar uma skill redireciona o comportamento completo do Hermes para o padrão definido nela.

🏗️ O que compõe uma skill

System prompt

Define como o Hermes pensa e responde enquanto a skill está ativa

Modelo específico

Cada skill pode usar um modelo diferente do padrão do config.json

Ferramentas disponíveis

Quais CLIs e capacidades estão acessíveis durante essa skill

Invocação on-demand

"Use o Orpheus para..." ativa a skill sem configuração adicional

🔑 Conceitos-chave

System prompt por skill

Instrução que redefine o comportamento do Hermes para um contexto específico

Modelo específico

Skills podem usar modelos diferentes — Orpheus usa Opus como Condutor

On-demand vs persistente

Skills invocadas por pedido vs skills sempre ativas em segundo plano

Módulos de comportamento

Skills como blocos de comportamento especializado que se ativam quando necessário

🏛️ O Pantheon — o painel de personas e skills

O Pantheon é a interface visual do Hermes para gerenciar personas. Sem o Pantheon, você editaria arquivos de configuração diretamente — mais propenso a erros e mais difícil de visualizar o que está ativo.

O que é

O Pantheon é a interface visual do Hermes acessada via hermes pantheon — um painel que lista todas as personas ativas, permite criar novas, editar system prompts e configurar modelos por persona sem editar arquivos JSON diretamente.

🖥️ Acessando o Pantheon

Via terminal:

hermes pantheon

Ou acesse pelo dashboard do Hermes → aba "Pantheon". Você verá todas as personas ativas, pode criar novas, editar system prompts e configurar modelos por persona.

🔑 Conceitos-chave

hermes pantheon

Comando para abrir a interface visual de gerenciamento de personas

Persona como entidade

Cada persona tem nome, missão, modelo e system prompt próprios

Visual vs arquivo

Pantheon é mais confiável que editar arquivos JSON diretamente

Sincronização

Mudanças no Pantheon são sincronizadas automaticamente com o Hermes

✨ Criando a persona Orpheus no Pantheon

Criar o Orpheus é o momento em que o sistema Triad se torna uma entidade nomeada dentro do Hermes. Dar um nome à persona ajuda a comunicar a intenção — "preciso do Orpheus" sinaliza claramente trabalho profundo que exige o fluxo completo.

O que é

O processo de criar a persona Orpheus no Pantheon — definindo nome, missão ("deep work agent com fluxo Triad completo") e Claude Opus 4 como modelo orquestrador, tornando o sistema Triad invocável por nome.

🛠️ Configuração da persona Orpheus

1.No Pantheon, clique em "Add Persona"
2.Nome: Orpheus
3.Descrição: "Deep work agent — usa o fluxo Triad para tarefas estratégicas complexas"
4.Modelo orquestrador: Claude Opus 4
5.Salvar — Orpheus aparece no Pantheon e pode ser invocado pelo nome

💡 Por que "Orpheus"

Orpheus é o personagem mitológico que desceu ao submundo para recuperar algo valioso — uma metáfora para trabalho profundo que vai além da superfície. O nome comunica ao Hermes (e a você) a natureza da tarefa: não é uma pergunta simples, é um trabalho que merece profundidade total do Triad.

🔑 Conceitos-chave

Add persona no Pantheon

Processo de criação de uma nova persona especializada

Nome Orpheus

Identidade que sinaliza profundidade e fluxo Triad completo

Modelo orquestrador

Claude Opus 4 como Condutor do fluxo Triad da persona

Invocação por nome

"Use o Orpheus para..." ativa o fluxo Triad completo automaticamente

📝 Os três system prompts do Triad — Conductor, Worker, Critic

O sistema Orpheus usa três system prompts distintos, um para cada papel. Esses são os prompts exatos que definem o comportamento do Triad — entendê-los permite ajustá-los para seus casos de uso específicos.

O que é

Os três system prompts que definem o comportamento de cada papel no Triad — Conductor (interroga, brifa, valida), Worker (executa de múltiplos ângulos, itera) e Critic (SHIP/REVISE/FUNDAMENTAL FLAW) — que são inseridos na persona Orpheus do Pantheon.

🎯 Conductor (Claude Opus 4)

Você é o Condutor do Hermes Triad.

1. INTERROGAR — faça 5-10 perguntas de clarificação até ter objetivo, restrições e critérios de sucesso.
2. BRIEFAR — escreva um briefing de uma página para o Worker.
3. VALIDAR — quando Worker + Crítico convergirem, valide contra o briefing.

Nunca execute o trabalho você mesmo.

O Conductor é o estrategista — define o que fazer, não faz.

⚡ Worker (DeepSeek V4)

Você é o Worker do Hermes Triad.

Receba o briefing do Conductor.
Ataque-o de 3-5 ângulos em paralelo.
Passe cada draft ao Critic.
Receba o feedback. Revise.
Se FUNDAMENTAL FLAW, reinicie com premissas invertidas.
Repita até SHIP.

O Worker é o executor — produz múltiplos ângulos e itera até convergir.

🔍 Critic (GPT)

Você é o Critic do Hermes Triad.

Para cada draft, retorne:
VERDITO (SHIP / REVISE / FUNDAMENTAL FLAW)
O QUE QUEBRA — falhas específicas com exemplos
O QUE FALTA — gaps vs briefing
MELHOR ÂNGULO — se múltiplos drafts, qual vence e por quê

Seja específico. Só aprove com SHIP quando o draft responde genuinamente ao briefing.

O Critic é o árbitro — aprova, rejeita ou descarta com justificativa específica.

🔑 Conceitos-chave

Conductor nunca executa

O papel do Conductor é estratégico — define e valida, não produz

Worker usa 3-5 ângulos

Múltiplas abordagens em paralelo para cobrir o espaço de soluções

SHIP/REVISE/FUNDAMENTAL FLAW

Três estados possíveis do Critic — critério claro de progresso

Critério de aprovação rigoroso

Critic só aprova SHIP quando o draft genuinamente responde ao briefing

🔄 Sincronizando Orpheus com o Hermes via prompt

Após criar o Orpheus no Pantheon, sincronize com o Hermes via prompt de contexto. A sincronização confirma que o Hermes entendeu o fluxo antes de você investir tempo em uma tarefa real — detecta gaps antes que importem.

O que é

Um prompt explícito que descreve o fluxo Triad completo ao Hermes e pede confirmação de entendimento — verificação prévia que detecta configurações incompletas antes de você investir tempo em uma tarefa real.

💬 Prompt de sincronização

"Hermes, estou sincronizando a persona Orpheus com você. Orpheus é meu agente de deep work que usa o fluxo Triad: Opus como Condutor (interroga, brifa, valida), DeepSeek como Worker (executa de 3-5 ângulos, loop com Crítico), GPT como Crítico (SHIP/REVISE/FUNDAMENTAL FLAW). Confirme que você entendeu o fluxo e me diga o que precisa antes de receber uma tarefa para Orpheus."

O Hermes deve confirmar o entendimento e solicitar qualquer informação que ainda precise para executar o fluxo corretamente.

🔑 Conceitos-chave

Prompt de sincronização

Confirmação explícita de que o Hermes entende o fluxo antes de usá-lo

Confirmação de entendimento

Hermes "lê o manual em voz alta" — valida sua compreensão

Detecção de gaps

Identifica o que falta antes de investir tempo em uma tarefa real

Iteração de configuração

O feedback do Hermes guia ajustes na configuração antes do primeiro uso

🧪 Testando Orpheus com uma tarefa real

O primeiro teste real deve ser uma tarefa de médio valor estratégico — importante o suficiente para valer o fluxo Triad, mas não crítica ao ponto de que um resultado imperfeito seja prejudicial. Use o primeiro teste para calibrar, não para decidir.

O que é

A execução do fluxo Triad completo com o Orpheus em uma tarefa estratégica real — passando por interrogação, briefing, execução em múltiplos ângulos, avaliação do Crítico e validação final do Conductor.

🎯 Tarefa de teste recomendada

"Orpheus, quero entender quais são os 3 principais erros que empreendedores cometem ao usar IA nos primeiros 90 dias. Me dê sua análise mais profunda."

O que você verá no fluxo:

1.Conductor (Opus) faz 5-8 perguntas de clarificação
2.Você responde às perguntas
3.Conductor escreve o briefing de uma página
4.Worker (DeepSeek) ataca de 3-5 ângulos diferentes
5.Critic (GPT) avalia com SHIP/REVISE/FUNDAMENTAL FLAW
6.Worker revisa com base no feedback
7.Conductor valida o resultado final contra o briefing

🔑 Conceitos-chave

Tarefa de médio valor

Importante para valer o esforço, mas não crítica se o resultado for imperfeito

Fluxo ponta a ponta

Testar o ciclo completo: interrogação → briefing → execução → crítica → validação

Interrogação como primeira etapa

Se o Orpheus pula a interrogação e vai direto para resposta, o Conductor está errado

Resultado convergido

O Triad deve convergir para uma resposta SHIP antes de apresentar o resultado

🔧 Ajustando e refinando a persona após o primeiro uso

O primeiro uso raramente é perfeito — e isso é esperado. A capacidade de identificar onde ajustar é o que separa quem tem um sistema funcional de quem tem um sistema excelente.

O que é

A avaliação pós-uso de cada papel separadamente — verificando se o Conductor fez perguntas específicas, se o Worker produziu ângulos genuinamente distintos e se o Critic foi objetivo — e aplicando ajustes cirúrgicos nos system prompts.

🔍 O que avaliar após o primeiro uso

O Conductor fez as perguntas certas?

Se as perguntas foram vagas ou genéricas demais:

Ajuste: adicione "cada pergunta deve revelar uma restrição ou pressuposto específico"

O Worker produziu ângulos genuinamente distintos?

Se os ângulos foram variações do mesmo tema:

Ajuste: adicione "cada ângulo deve usar uma premissa fundamentalmente diferente"

O Critic foi específico o suficiente?

Se o feedback foi vago ou apenas positivo:

Ajuste: adicione "cite exemplos específicos do texto quando apontar falhas"

O Conductor foi muito verboso?

Se fez 15 perguntas quando 5 bastavam:

Ajuste: "máximo 5 perguntas, uma de cada vez, mais diretas"

🔑 Conceitos-chave

Avaliação pós-uso

Avaliar cada papel separadamente — Conductor, Worker e Critic podem ter problemas distintos

Ajuste de system prompt

Mudanças cirúrgicas no prompt resolvem problemas específicos sem quebrar o que funciona

Iteração esperada

Primeiro uso imperfeito não é falha — é calibração necessária do sistema

Critérios por papel

Cada papel tem critérios de qualidade diferentes — avalie com critérios corretos
