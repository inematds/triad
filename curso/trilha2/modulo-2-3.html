⚡ Por que ter chave própria da DeepSeek no OpenRouter

Por padrão, chamadas ao DeepSeek V4 via OpenRouter compartilham rate limits com outros usuários. Para uso intensivo em loops overnight do Triad, isso causa erros 429 que interrompem o trabalho no meio. BYOK é a proteção estrutural contra esse problema.

O que é

O BYOK (Bring Your Own Key) é a funcionalidade do OpenRouter que permite vincular sua chave própria da DeepSeek Platform — dando rate limits independentes e custos cobrados diretamente do provedor, não do pool compartilhado.

✗ Sem chave própria

  • •Rate limit compartilhado com todos os usuários do OpenRouter
  • •Erro 429 em picos de uso
  • •Loop overnight pode ser interrompido

✓ Com chave própria (BYOK)

  • •Rate limit dedicado e independente
  • •Loops overnight sem interrupção
  • •Custo cobrado diretamente da DeepSeek Platform

🔑 Conceitos-chave

Rate limit compartilhado

Limite dividido entre todos os usuários do OpenRouter sem chave própria

429 Too Many Requests

Erro que interrompe loops quando o rate limit é excedido

BYOK como solução

Bring Your Own Key — chave própria que dá rate limits independentes

Continuidade de loop

Loops overnight precisam rodar sem interrupção por rate limits

💳 Criando conta e carregando créditos na DeepSeek Platform

Acesse platform.deepseek.com, crie uma conta com email e adicione créditos. $5-$10 é suficiente para semanas de uso normal do Triad — o custo por chamada do DeepSeek V4 é significativamente menor que outros modelos de alta capacidade.

O que é

A plataforma oficial da DeepSeek em platform.deepseek.com onde você cria uma conta separada do OpenRouter, carrega créditos diretamente e gerencia suas chaves de API da DeepSeek.

📋 Processo de criação de conta

1.Acesse platform.deepseek.com
2.Crie conta com email e confirme
3.Vá em "Billing" → "Top Up" e adicione $5-$10 via cartão
4.O saldo aparece em tempo real no dashboard da plataforma

💡 Saldo DeepSeek ≠ Saldo OpenRouter

O saldo que você adiciona na DeepSeek Platform é separado do saldo no OpenRouter. Quando você usa BYOK, os créditos são consumidos diretamente da DeepSeek Platform — não do OpenRouter. Mantenha saldo em ambos se usar modelos de outros provedores pelo OpenRouter também.

🔑 Conceitos-chave

platform.deepseek.com

Plataforma oficial da DeepSeek para conta, créditos e chaves de API

$5-$10 inicial

Suficiente para semanas de uso do Triad com DeepSeek como Worker

Saldos separados

DeepSeek Platform e OpenRouter têm saldos independentes

Billing em tempo real

O dashboard mostra consumo imediatamente após cada chamada

🗝️ Gerando API key da DeepSeek

Na DeepSeek Platform, gere sua chave de API para uso no BYOK do OpenRouter. A chave só é exibida uma vez — copie imediatamente para um local seguro antes de fechar a janela.

O que é

Uma string de autenticação com prefixo sk- gerada na seção "API Keys" da DeepSeek Platform — exibida apenas uma vez no momento da criação e necessária para ativar o BYOK no OpenRouter.

🔐 Gerando a chave

1.Vá em "API Keys" → "Create new API key"
2.Dê um nome descritivo: hermes-triad-worker
3.A chave gerada começa com sk-
4.Copie imediatamente para gerenciador de senhas
5.A chave não será exibida novamente após fechar a modal

🔑 Conceitos-chave

sk- prefix

Prefixo padrão das chaves de API da DeepSeek Platform

Gerenciador de senhas

1Password, Bitwarden ou similar — nunca em arquivo de texto simples

Variável de ambiente

Alternativa ao gerenciador — armazene como DEEPSEEK_API_KEY no sistema

Backup seguro

A chave não pode ser recuperada — perder significa gerar nova e reconfigurar

4

🔗 Adicionando a chave no OpenRouter (Bring Your Own Keys)

A integração BYOK acontece uma única vez e funciona para todas as chamadas futuras. Não é necessário mudar nada no Hermes — ele continua usando a mesma API key do OpenRouter, mas o OpenRouter internamente usa sua chave da DeepSeek.

O que é

O processo de vincular sua chave da DeepSeek ao OpenRouter via "Settings → Integrations → Bring Your Own Keys" — configuração transparente para o Hermes que redireciona chamadas ao DeepSeek para a sua chave própria.

🛠️ Configurando BYOK no OpenRouter

1.No dashboard do OpenRouter, vá em "Settings" → "Integrations"
2.Clique em "Bring Your Own Keys"
3.Procure por "DeepSeek" na lista de provedores
4.Clique em "Configure" e cole sua chave da DeepSeek Platform
5.O OpenRouter valida automaticamente — aguarde confirmação verde
6.Salvar — a partir de agora todas as chamadas ao deepseek/deepseek-chat usam sua chave

💡 Transparência para o Hermes

O Hermes não precisa saber que você está usando BYOK. Ele continua chamando o OpenRouter com a mesma API key — o OpenRouter é quem redireciona para a DeepSeek Platform usando sua chave. Essa transparência significa zero reconfiguração no Hermes após ativar o BYOK.

🔑 Conceitos-chave

openrouter.ai/settings

URL onde as configurações de BYOK são gerenciadas

Validação automática

OpenRouter faz uma chamada de teste para confirmar que a chave funciona

Transparência para o Hermes

BYOK é invisível para o Hermes — zero reconfiguração necessária

Uma configuração

Configure uma vez, todas as chamadas futuras ao DeepSeek usam sua chave

5

🎯 Configurando DeepSeek como modelo Worker no Hermes

Em ~/.hermes/config.json, defina o papel Worker para usar o modelo correto. A configuração correta garante que o DeepSeek V4 executa o papel de execução — não um modelo mais caro sendo usado onde DeepSeek basta.

O que é

A entrada "worker_model": "deepseek/deepseek-chat" no config.json do Hermes, com os parâmetros de temperature e max_tokens que definem o comportamento do Worker nas execuções do Triad.

📝 Configuração no config.json

Modelo do Worker

"worker_model": "deepseek/deepseek-chat"

Parâmetros recomendados

"worker_temperature": 0.7, "worker_max_tokens": 4096

temperature 0.7 = criatividade controlada para múltiplos ângulos distintos

🔑 Conceitos-chave

deepseek/deepseek-chat

Nome exato do modelo no OpenRouter para o Worker do Triad

temperature 0.7

Balanceia criatividade e coerência para ângulos distintos no Worker

max_tokens

Limite de tokens por resposta — controla custo e tamanho do output

Override por skill

A skill Orpheus pode sobrescrever esse padrão para casos específicos

6

🧪 Testando o Worker — tarefa simples ponta a ponta

Teste o Worker com uma tarefa estruturada que verifica os três requisitos: modelo correto chamado, ângulos distintos produzidos, custo adequado. Testar o Worker isoladamente antes do Triad completo é mais fácil de diagnosticar quando algo falha.

O que é

Um teste isolado do papel Worker — pedindo explicitamente ao DeepSeek V4 para produzir três ângulos distintos sobre uma questão estratégica, e verificando nos logs se o modelo correto foi chamado e o custo está dentro do esperado.

🔍 Tarefa de teste recomendada

"Usando o modelo DeepSeek V4 como Worker, liste 3 ângulos diferentes para responder à pergunta: qual é a principal vantagem competitiva de usar múltiplos modelos de IA?"

O que verificar:

  • ✓Log mostra DeepSeek como provedor (não Claude)
  • ✓Três ângulos genuinamente distintos (não variações do mesmo)
  • ✓Custo significativamente menor que usando Opus

🔑 Conceitos-chave

Teste isolado

Testar componentes individualmente antes de integrá-los ao loop completo

Verificação de logs

Confirmar que o modelo correto foi chamado — não assuma, verifique

Ângulos distintos

Cada ângulo deve usar uma premissa diferente — não variações do mesmo

Custo comparativo

DeepSeek deve custar uma fração do Opus para a mesma tarefa

7

📊 Monitorando uso e custo no primeiro ciclo real

Após a primeira execução completa do Triad com DeepSeek como Worker, monitore o dashboard para calibrar expectativas e confirmar que o BYOK está ativo. O monitoramento do primeiro ciclo fecha o loop de configuração — você sabe que tudo funciona corretamente.

O que é

A verificação pós-execução no dashboard do OpenRouter que confirma o BYOK ativo, filtra o custo específico das chamadas ao DeepSeek e estabelece o custo por execução como benchmark para uso futuro.

📈 O que monitorar no primeiro ciclo

No OpenRouter dashboard

Filtre as chamadas por modelo deepseek/deepseek-chat e some o custo total do ciclo

Verificar BYOK ativo

O log mostra "provider: DeepSeek via BYOK" quando sua chave é usada — não o pool compartilhado

Comparar com custo sem DeepSeek

Estime o custo equivalente se o Opus executasse o trabalho do Worker — a diferença é o valor do BYOK

Calcular custo por execução

Divida o custo total pelo número de loops — saber o custo por tarefa calibra o uso futuro

🔑 Conceitos-chave

Filtro por modelo

Isola o custo do Worker DeepSeek dos outros modelos do ciclo

Verificação BYOK ativa

Confirma que sua chave está sendo usada, não o pool compartilhado

Custo por execução

Métrica que permite decidir quando vale usar o Triad completo vs modelo único

Calibragem de expectativas

Primeiro ciclo define o benchmark de custo para uso futuro
