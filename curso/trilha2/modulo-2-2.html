🌐 Criando sua conta no OpenRouter

A conta do OpenRouter é o gateway para todos os modelos do Triad. Sem ela, você está limitado aos modelos de um único provedor — o que elimina o principal benefício arquitetural do sistema.

O que é

O OpenRouter é uma plataforma que agrega modelos de múltiplos provedores — Anthropic, DeepSeek, OpenAI e outros — em uma única API unificada. Uma conta é necessária para acessar esses modelos via Hermes.

📋 Processo de criação de conta

1.Acesse openrouter.ai e clique em "Sign In"
2.Crie conta com email ou via OAuth (Google ou GitHub)
3.Confirme o email se usar cadastro direto
4.Dashboard com créditos de teste fica disponível imediatamente
5.Adicione método de pagamento para uso regular

🔑 Conceitos-chave

openrouter.ai

Plataforma que agrega modelos de múltiplos provedores em uma API unificada

OAuth de conta

Login via Google ou GitHub — mais rápido e sem senha a gerenciar

Créditos de teste

Valor inicial para testar sem precisar adicionar cartão imediatamente

Dashboard inicial

Centro de controle de uso, custos, chaves e modelos disponíveis

🗝️ Gerando a API key principal

A API key é o único elemento que autentica suas chamadas ao OpenRouter. Perdê-la significa gerar uma nova e atualizar todas as integrações — gerenciar chaves corretamente desde o início é segurança básica.

O que é

Uma string de autenticação com prefixo sk-or- gerada no dashboard do OpenRouter que autoriza todas as chamadas de API do Hermes ao sistema de modelos.

🔐 Gerando a chave corretamente

1.No dashboard do OpenRouter, vá em "Keys" → "Create Key"
2.Dê um nome descritivo: hermes-triad
3.Defina um limite de crédito (ex: $20) para proteção contra uso acidental
4.Copie a chave imediatamente — começa com sk-or-
5.Salve em gerenciador de senhas — não será exibida novamente

⚠️ Nunca fazer

  • ✗Commitar a chave em repositórios git — use variáveis de ambiente
  • ✗Compartilhar a chave com terceiros sem criar uma chave separada para eles
  • ✗Deixar a chave sem limite de crédito — um loop infinito pode gerar custos inesperados

🔑 Conceitos-chave

sk-or- prefix

Prefixo que identifica chaves do OpenRouter vs outros provedores

Limite de crédito

Teto de gasto por chave — proteção contra uso acidental excessivo

Armazenamento seguro

Gerenciador de senhas ou variável de ambiente no sistema

Rotação de chaves

Prática de substituir chaves periodicamente por segurança

⚙️ Configurando OpenRouter no Hermes

Com a API key em mãos, configure o OpenRouter como provedor no Hermes via wizard interativo. Erros nessa etapa causam falhas silenciosas onde o Hermes usa o modelo errado sem avisar — o wizard previne isso.

O que é

O processo de vincular a API key do OpenRouter ao Hermes via hermes setup model — associando o provedor e testando a conexão antes de salvar a configuração.

🛠️ Passo a passo do setup

hermes setup model

Abre o wizard de configuração de modelos com lista de provedores suportados.

  1. 2.Use barra de espaço para selecionar "OpenRouter" na lista
  2. 3.Pressione Enter para confirmar
  3. 4.Cole sua API key quando solicitado (começa com sk-or-)
  4. 5.Hermes testa a conexão automaticamente — aguarde confirmação
  5. 6.Configure modelos padrão para cada papel: Condutor, Executor, Crítico

🔑 Conceitos-chave

hermes setup model

Wizard de configuração de provedores e modelos padrão

Teste de conexão automático

Hermes valida a chave com uma chamada real antes de salvar

Modelo padrão por papel

Condutor, Executor e Crítico podem usar modelos diferentes

Falhas silenciosas

Configuração errada usa modelo incorreto sem avisar — o wizard previne isso

🧪 Testando a conexão — chamada de verificação

Após configurar o OpenRouter, teste com uma chamada simples que verifica o provedor nos logs. Uma falha de conexão aqui é fácil de diagnosticar — muito mais do que uma falha no meio de um loop complexo do Triad.

O que é

Uma chamada de smoke test que verifica se o OpenRouter responde corretamente via Hermes — checando autenticação, nome de modelo e log de provedor antes de usar o sistema em tarefas reais.

🔍 Chamada de teste recomendada

hermes "Use o modelo deepseek/deepseek-chat via OpenRouter e me diga qual é a capital do Brasil"

O que verificar na resposta:

  • ✓Resposta menciona "Brasília"
  • ✓Log mostra provedor "OpenRouter" ou "DeepSeek"
  • ✓Nenhum erro de autenticação ou rate limit

❌ Erros e soluções

401

Chave errada ou sem créditos → verifique a key e adicione créditos no dashboard

404

Nome de modelo incorreto → verifique o nome exato no dashboard de modelos do OpenRouter

429

Rate limit → adicione chave própria do DeepSeek via BYOK (próximo módulo)

🔑 Conceitos-chave

Teste de smoke

Chamada mínima que confirma que o sistema funciona end-to-end

deepseek/deepseek-chat

Nome exato do modelo no OpenRouter para o Worker do Triad

Log de provedor

Informação nos logs que confirma qual modelo e provedor foi usado

Diagnóstico de autenticação

401 = chave, 404 = modelo, 429 = rate limit, timeout = velocidade

📊 Navegando o dashboard — uso, custos, modelos disponíveis

O dashboard do OpenRouter é o painel de controle financeiro do seu sistema de IA. Monitorar regularmente é a única forma de identificar uso inesperado, custos fora do esperado ou chamadas a modelos errados.

O que é

A interface em openrouter.ai/activity que exibe o histórico completo de chamadas, custo por chamada, tokens consumidos, saldo de créditos e alertas de gasto configuráveis.

🖥️ O que você encontra em openrouter.ai/activity

Histórico de chamadas

  • •Modelo usado em cada chamada
  • •Tokens de entrada e saída
  • •Custo por chamada em dólares
  • •Timestamp e status da chamada

Controle financeiro

  • •Gráfico de uso por período
  • •Custo acumulado no mês
  • •Saldo de créditos restante
  • •Filtro por chave de API

🔑 Conceitos-chave

openrouter.ai/activity

URL direta para o histórico de chamadas e custos

Custo por chamada

Granularidade que permite identificar chamadas caras ou inesperadas

Filtro de modelos

Aba "Models" com preços atualizados e capacidades por modelo

Alerta de custo

Configure notificações quando o gasto atingir um teto definido

🎯 Selecionando modelos padrão para cada papel do Triad

Configurar os modelos corretos por papel é o que transforma o Hermes em um sistema Triad real. Sem essa configuração, todos os papéis usam o mesmo modelo padrão — eliminando o benefício arquitetural do sistema multi-modelo.

O que é

A configuração no ~/.hermes/config.json que associa cada papel do Triad a um modelo específico — Claude Opus 4 para Condutor, DeepSeek para Worker e GPT-4o para Crítico.

🗂️ Configuração em ~/.hermes/config.json

Condutor anthropic/claude-opus-4

Modelo mais capaz — interroga, brifa e valida. Use claude-opus-4:nitro para velocidade máxima.

Executor / Worker deepseek/deepseek-chat

Alta performance a baixo custo — executa de 3-5 ângulos. Configure BYOK no próximo módulo para rate limits independentes.

Crítico openai/gpt-4o

Perspectiva diferente da Anthropic — retorna SHIP/REVISE/FUNDAMENTAL FLAW. Use openai/chatgpt-4o-latest para versão mais recente.

🔑 Conceitos-chave

config.json

Arquivo central onde os modelos por papel são definidos

Sufixos :nitro/:floor

:nitro para velocidade máxima, :floor para custo mínimo

Override por skill

Cada skill pode sobrescrever o modelo padrão do config.json

anthropic/ vs openai/ vs deepseek/

Prefixo que identifica o provedor do modelo no OpenRouter

🔧 Troubleshooting — erros comuns e soluções

Os cinco erros mais comuns ao configurar o OpenRouter no Hermes têm causas simples e soluções diretas. Saber os erros antes de encontrá-los reduz o tempo de resolução de horas para minutos.

O que é

Os cinco erros mais frequentes na configuração do OpenRouter — 401, 404, 429, timeout e config não salva — com causa específica e solução direta para cada um.

401 Unauthorized Autenticação falhou

Causa: Chave errada ou expirada, ou saldo zero na conta

Solução: Verifique se colou a chave corretamente; adicione créditos no dashboard do OpenRouter

404 Model not found Modelo não existe

Causa: Nome de modelo incorreto ou desatualizado

Solução: Verifique o nome exato na aba "Models" do dashboard do OpenRouter

429 Rate limit Limite de requisições atingido

Causa: Sem chave própria, usando rate limit compartilhado do OpenRouter

Solução: Adicione sua chave própria da DeepSeek via BYOK (Módulo 2.3)

Timeout Chamada não retornou a tempo

Causa: Modelo lento ou instância sobrecarregada

Solução: Adicione sufixo :nitro ao nome do modelo para roteamento prioritário

Config não salva Configuração perdida após reiniciar

Causa: Wizard interrompido antes de salvar, ou permissão de escrita

Solução: Edite ~/.hermes/config.json diretamente e reinicie o Hermes

🔑 Conceitos-chave

401 vs 404 vs 429

Cada código HTTP indica uma causa diferente e exige solução diferente

BYOK como solução de rate limit

Chave própria dá rate limits independentes da base compartilhada

:nitro para timeout

Sufixo que roteia para instâncias mais rápidas do modelo

config.json direto

Edição manual como fallback quando o wizard falha
