⚙️ Configurando o ambiente Hermes

Instalação passo a passo, configuração de diretórios e verificação de saúde do sistema antes de avançar.

O que é:

Para instalar e usar o Hermes Agent você precisa de: sistema operacional macOS, Linux ou Windows com WSL; Node.js 18+ instalado; conta na Anthropic (para Claude); terminal com permissões de usuário; e opcionalmente VS Code ou outro editor de código. Não é necessário ser desenvolvedor — mas familiaridade com o terminal é essencial.

Por que aprender:

Verificar pré-requisitos antes de instalar evita erros frustrantes no meio do processo. Saber o que você tem e o que falta é o primeiro passo para uma instalação sem surpresas.

Conceitos-chave:

Node.js como runtime · Terminal/shell básico · Conta Anthropic · WSL para Windows · Permissões de usuário vs admin

O que é:

O Hermes Agent é instalado via terminal. O processo envolve: clonar ou instalar via npm o pacote do Hermes, configurar variáveis de ambiente, rodar o setup inicial que guia pela configuração de modelos. O comando central é hermes setup que abre um wizard interativo para configurar chaves de API e preferências.

Por que aprender:

Seguir a sequência correta de instalação evita estados inconsistentes. Pular passos do setup inicial é a causa mais comum de erros difíceis de diagnosticar.

Conceitos-chave:

npm install global · hermes setup wizard · Variáveis de ambiente · Estado de configuração · Verificação pós-instalação

O que é:

Após a instalação base, o Hermes cria uma estrutura de diretórios em ~/.hermes/ que inclui: soul.md (contexto pessoal), skills/ (personas e skills), memory/ (histórico e memória) e config.json (configurações gerais). As variáveis de ambiente necessárias são: ANTHROPIC_API_KEY, e opcionalmente OPENROUTER_API_KEY para modelos alternativos.

Por que aprender:

Conhecer a estrutura de diretórios do Hermes permite fazer backup do que importa, entender onde fica cada coisa e diagnosticar problemas com mais rapidez.

Conceitos-chave:

~/.hermes/ como base · Estrutura de diretórios · ANTHROPIC_API_KEY · Backup de configuração · config.json

O que é:

Após a instalação e configuração, o primeiro teste é uma conversa simples: perguntar ao Hermes algo sobre você mesmo para confirmar que ele responde, acessa o soul.md e executa ferramentas básicas. Um bom teste inicial é: "Hermes, o que você sabe sobre mim até agora?" seguido de "Use o terminal para criar um arquivo de teste em ~/Desktop/hermes-test.txt".

Por que aprender:

Testar uma funcionalidade simples antes de avançar para configurações complexas isola problemas. Se o básico funciona, a instalação está correta.

Conceitos-chave:

Smoke test · Verificação de soul.md · Execução de ferramenta de terminal · Iteração de diagnóstico

O que é:

O Hermes tem uma interface de dashboard (acessível via hermes dashboard ou pela extensão do VS Code/IDE) que centraliza: histórico de conversas, skills e personas ativas (Pantheon), uso e custo de tokens, status de jobs agendados e memória ativa. O dashboard é onde você gerencia o sistema visualmente sem precisar do terminal.

Por que aprender:

Conhecer o dashboard antes de precisar dele reduz fricção. Você sabe onde olhar quando algo vai errado e como navegar entre as diferentes funcionalidades do Hermes.

Conceitos-chave:

hermes dashboard · Pantheon visual · Histórico de sessões · Status de jobs · Custo acumulado

O que é:

Claude Code (CLI de desenvolvimento) e Hermes (agente pessoal) podem ser usados de forma integrada. O Hermes pode invocar Claude Code como uma ferramenta para tarefas de código, e o dashboard do Hermes pode mostrar dados de uso do Claude Code. A integração é configurada via hermes setup integrations e permite que Hermes delegue tarefas de código sem sair do seu fluxo principal.

Por que aprender:

A integração entre Hermes e Claude Code é onde o sistema atinge seu potencial máximo — Hermes orquestra a vida toda, incluindo o trabalho de código que Claude Code executa.

Conceitos-chave:

hermes setup integrations · Delegação de tarefas de código · Dashboard unificado · Fluxo de orquestração

O que é:

Após completar a instalação, um checklist de verificação de saúde confirma que tudo está funcionando: Hermes responde ✓ | soul.md é carregado ✓ | Chave API está ativa ✓ | Terminal funciona como ferramenta ✓ | Dashboard abre ✓ | Histórico é salvo entre sessões ✓ | Skills podem ser carregadas ✓.

Por que aprender:

Um sistema parcialmente configurado pode parecer funcionar mas falhar em casos importantes. O checklist sistemático garante que você tem uma base sólida antes de avançar.

Conceitos-chave:

Verificação sistemática · Diagnóstico de instalação · Base sólida antes de avançar · Identificação de problemas latentes

🔑 Conectando OpenRouter

Chaves, dashboard e roteamento — o gateway para todos os modelos do Triad em um único lugar.

O que é:

Acesse openrouter.ai, clique em "Sign In" e crie uma conta com email ou via OAuth (Google, GitHub). Após confirmar o email, você tem acesso ao dashboard com créditos de teste. Para uso regular, você precisará adicionar créditos ou um método de pagamento. O processo completo leva menos de 5 minutos.

Por que aprender:

A conta do OpenRouter é o gateway para todos os modelos do Triad. Sem ela, você está limitado aos modelos de um único provedor.

Conceitos-chave:

openrouter.ai · OAuth de conta · Créditos de teste · Método de pagamento · Dashboard inicial

O que é:

No dashboard do OpenRouter, vá em "Keys" → "Create Key". Dê um nome descritivo (ex: "hermes-triad"), defina um limite de crédito opcional para controle de gastos e copie a chave gerada (começa com sk-or-). Armazene em local seguro — não é possível ver a chave novamente após fechar a janela.

Por que aprender:

A API key é o único elemento que autentica suas chamadas. Perdê-la significa gerar uma nova e atualizar todas as integrações. Gerenciar chaves corretamente é segurança básica.

Conceitos-chave:

sk-or- prefix · Limite de crédito como proteção · Armazenamento seguro · Rotação de chaves · Nunca commitar no git

O que é:

No terminal, execute hermes setup model. O wizard mostrará uma lista de provedores suportados. Use a barra de espaço para selecionar "OpenRouter" e pressione Enter. Cole sua API key quando solicitado. O Hermes confirma a conexão testando uma chamada simples. Você pode configurar qual modelo OpenRouter usar como padrão para cada papel (Condutor, Executor, Crítico).

Por que aprender:

O setup correto garante que o Hermes roteie chamadas pelo OpenRouter automaticamente. Erros nessa etapa causam falhas silenciosas onde Hermes usa o modelo errado ou não acessa modelos alternativos.

Conceitos-chave:

hermes setup model · Seleção de provedor · Teste de conexão automático · Modelo padrão por papel · Verificação de configuração

O que é:

Após configurar OpenRouter, teste com: hermes "Use o modelo deepseek/deepseek-chat via OpenRouter e me diga qual é a capital do Brasil". Se a resposta mencionar "Brasília" e o log mostrar o provedor OpenRouter/DeepSeek, a conexão está funcionando. Se houver erro de autenticação, verifique se a chave foi colada corretamente e se tem créditos disponíveis.

Por que aprender:

Testar a conexão com um caso simples antes de rodar tarefas complexas economiza tempo de debugging. Uma falha aqui é infinitamente mais fácil de diagnosticar do que uma falha no meio de um loop do Triad.

Conceitos-chave:

Teste de smoke · deepseek/deepseek-chat · Log de provedor · Diagnóstico de autenticação · Verificação de créditos

O que é:

O dashboard do OpenRouter (openrouter.ai/activity) mostra: histórico de chamadas com modelo, tokens e custo de cada uma; gráfico de uso por período; custo acumulado; e lista de todos os modelos disponíveis com preços atualizados. A aba "Models" permite filtrar por capacidade, preço e provedor.

Por que aprender:

Monitorar o dashboard regularmente é a única forma de identificar uso inesperado, custos fora do esperado ou chamadas a modelos errados. É o painel de controle financeiro do seu sistema de IA.

Conceitos-chave:

openrouter.ai/activity · Custo por chamada · Gráfico de uso · Filtro de modelos · Alerta de custo

O que é:

No Hermes, você configura qual modelo OpenRouter usar para cada papel: Condutor → anthropic/claude-opus-4 (ou claude-opus-4:nitro para velocidade), Executor → deepseek/deepseek-chat, Crítico → openai/gpt-4o. Essa configuração fica em ~/.hermes/config.json e pode ser sobrescrita por skill específica.

Por que aprender:

Configurar os modelos corretos por papel é o que transforma o Hermes em um sistema Triad real. Sem essa configuração, todos os papéis usam o mesmo modelo padrão — eliminando o benefício arquitetural.

Conceitos-chave:

config.json · Modelo por papel · Sufixos :nitro/:floor · Override por skill · anthropic/ vs openai/ vs deepseek/

O que é:

Os erros mais comuns ao configurar OpenRouter no Hermes: (1) 401 Unauthorized — chave errada ou sem créditos; (2) 404 Model not found — nome de modelo incorreto; (3) Rate limit 429 — sem chave própria usando modelo popular, solução: adicionar BYOK; (4) Timeout — modelo lento, solução: adicionar sufixo :nitro; (5) Configuração não salva — editar config.json diretamente e reiniciar Hermes.

Por que aprender:

Saber os erros antes de encontrá-los reduz drasticamente o tempo de resolução. A maioria dos problemas de integração tem causa simples e solução direta.

Conceitos-chave:

401 vs 404 vs 429 · BYOK como solução de rate limit · :nitro para timeout · config.json direto · Reinicialização do Hermes

🤖 Integrando DeepSeek V4 como Worker do Triad

Chave própria, BYOK no OpenRouter e rate limits independentes para loops overnight sem interrupção.

O que é:

Por padrão, chamadas ao DeepSeek V4 via OpenRouter compartilham rate limits com outros usuários. Para uso intensivo — especialmente loops overnight do Triad — isso pode causar erros 429 (rate limit excedido). Com chave própria (BYOK), suas chamadas têm rate limits independentes da base de usuários do OpenRouter.

Por que aprender:

Rate limits inesperados no meio de um loop overnight podem interromper o trabalho e deixar uma tarefa incompleta. BYOK é a proteção contra esse problema.

Conceitos-chave:

Rate limit compartilhado vs dedicado · 429 Too Many Requests · BYOK como solução · Uso intensivo overnight · Continuidade de loop

O que é:

Acesse platform.deepseek.com, crie uma conta com email e confirme. Na seção "Billing" ou "Top Up", adicione créditos via cartão de crédito. Para começar, $5-$10 é suficiente para semanas de uso normal do Triad. A plataforma mostra saldo em tempo real e histórico de uso por modelo.

Por que aprender:

Os créditos na DeepSeek Platform são consumidos apenas quando você usa sua chave própria. Sem créditos, a chave retorna erro de pagamento. É importante entender que o saldo na DeepSeek é separado do saldo no OpenRouter.

Conceitos-chave:

platform.deepseek.com · Top up de créditos · Saldo DeepSeek vs OpenRouter · $5-$10 inicial · Billing em tempo real

O que é:

Na DeepSeek Platform, vá em "API Keys" → "Create new API key". Dê um nome (ex: "hermes-triad-worker"), copie a chave gerada (começa com sk-). Como toda chave de API, você não consegue visualizá-la novamente depois — copie imediatamente para um gerenciador de senhas ou variável de ambiente.

Por que aprender:

A chave da DeepSeek é o que vincula suas chamadas ao seu saldo na plataforma. Sem ela, você depende do pool compartilhado do OpenRouter com seus rate limits.

Conceitos-chave:

platform.deepseek.com/api-keys · sk- prefix · Gerenciador de senhas · Variável de ambiente · Backup seguro

O que é:

No dashboard do OpenRouter, vá em "Settings" → "Integrations" → "Bring Your Own Keys". Procure por "DeepSeek" na lista de provedores, clique em "Configure" e cole sua chave da DeepSeek Platform. O OpenRouter valida a chave automaticamente. Após salvar, chamadas ao deepseek/deepseek-chat via OpenRouter usarão automaticamente sua chave.

Por que aprender:

A integração BYOK acontece uma única vez e funciona para todas as chamadas futuras. Não é necessário mudar nada no Hermes após essa configuração — ele continua usando a mesma API key do OpenRouter.

Conceitos-chave:

openrouter.ai/settings · BYOK configuration · Validação automática · Transparência para o Hermes · Uma configuração, uso contínuo

O que é:

Em ~/.hermes/config.json, defina o papel Worker para usar o modelo deepseek/deepseek-chat. Na skill Orpheus, o system prompt do Worker especifica que ele deve usar esse modelo. Você pode também configurar parâmetros como temperature (0.7 recomendado) e max_tokens para controlar o tamanho das respostas.

Por que aprender:

A configuração correta do modelo Worker é o que garante que o DeepSeek V4 executa o papel correto no fluxo. Sem isso, o Hermes pode usar um modelo mais caro para execução quando DeepSeek é suficiente.

Conceitos-chave:

config.json Worker model · deepseek/deepseek-chat · temperature 0.7 · max_tokens · Override por skill

O que é:

Teste o Worker com uma tarefa estruturada simples: "Usando o modelo DeepSeek V4 como Worker, liste 3 ângulos diferentes para responder à pergunta: qual é a principal vantagem competitiva de usar múltiplos modelos de IA?". Verifique nos logs que o DeepSeek foi chamado (não Claude), que a resposta tem três ângulos distintos e que o custo foi significativamente menor que usar Opus.

Por que aprender:

Testar o Worker isoladamente antes de rodar o Triad completo garante que cada componente funciona. Problemas no Worker são mais fáceis de diagnosticar sozinhos do que dentro do loop completo.

Conceitos-chave:

Teste isolado de componente · Verificação de logs · Custo comparativo · Três ângulos como output esperado · Diagnóstico antes do loop completo

O que é:

Após a primeira execução completa do Triad com DeepSeek como Worker, monitore: no OpenRouter dashboard, filtre as chamadas por modelo deepseek/deepseek-chat e veja o custo total; compare com o custo que seria pagar com Opus; calcule quantas execuções similares você consegue por dollar; e verifique se sua chave BYOK foi usada (o log mostra "provider: DeepSeek via BYOK" quando sua chave é ativa).

Por que aprender:

O monitoramento do primeiro ciclo real calibra suas expectativas de custo e confirma que a configuração BYOK está funcionando corretamente. É o fechamento do loop de configuração.

Conceitos-chave:

Filtro por modelo no dashboard · Custo comparativo Opus vs DeepSeek · Custo por execução · Verificação BYOK ativa · Calibragem de expectativas

🎬 Gemini CLI — Capacidade multimodal no terminal

Instale, autentique e integre o Gemini CLI ao Hermes para análise de vídeo, imagem e PDF diretamente do terminal.

O que é:

CLI significa Command Line Interface — uma forma de interagir com um serviço via terminal, sem interface gráfica. O Gemini CLI é a implementação oficial do Google que permite usar o Gemini diretamente no terminal. O que o torna especialmente poderoso é que, quando instalado localmente, o Hermes pode invocá-lo como uma ferramenta — transformando Gemini em um executor de tarefas multimodais disponível a qualquer momento.

Por que aprender:

Entender o que CLI significa e por que o Gemini CLI especificamente vale a pena instalar é o que justifica o esforço de configuração. Não é só mais um modelo — é capacidade multimodal local.

Conceitos-chave:

CLI vs interface gráfica · Local vs API remota · Hermes invocando CLI · Multimodal local · Custo zero com conta Google

O que é:

A instalação do Gemini CLI é feita via npm: npm install -g @google/gemini-cli. Alternativamente, você pode clonar o repositório oficial do GitHub e instalar localmente. Para confirmar a instalação, execute gemini --version. Se retornar o número da versão, o CLI está pronto.

Por que aprender:

A instalação via npm é o caminho mais simples e garante que você recebe atualizações automáticas. Clonar o repositório é para quem quer modificar o CLI ou usar versões de desenvolvimento.

Conceitos-chave:

npm install -g · gemini --version · Repositório GitHub oficial · Atualização automática · Instalação global vs local

O que é:

Após instalar, execute gemini auth login. O CLI abrirá um link de autenticação no browser ou solicitará que você cole um código de verificação. Autentique com sua conta Google. A autenticação é salva localmente e válida por um período longo. O tier gratuito do Gemini é generoso para uso pessoal e inclui acesso ao Gemini 2.0.

Por que aprender:

A autenticação via conta Google elimina a necessidade de chave de API — o que significa custo zero para começar. Entender o processo de auth evita confusão entre autenticação do CLI e chave de API.

Conceitos-chave:

gemini auth login · OAuth com conta Google · Tier gratuito · Sem API key necessária · Validade da autenticação

O que é:

Com o Gemini CLI autenticado, os comandos básicos são: gemini "sua pergunta aqui" para uma resposta simples; gemini -f arquivo.pdf "resuma este documento" para análise de arquivo; gemini -f imagem.png "descreva o que você vê" para análise visual. O CLI suporta arquivos de texto, PDF, imagens e vídeos como entrada.

Por que aprender:

Familiaridade com a sintaxe básica do CLI é necessária antes de integrá-lo ao Hermes. Saber o que funciona diretamente no terminal facilita o debugging quando a integração tem problemas.

Conceitos-chave:

gemini "prompt" · -f para arquivo · Tipos de arquivo suportados · Análise de PDF, imagem, vídeo · Saída no terminal

O que é:

Para que o Hermes possa usar o Gemini CLI, você precisa configurá-lo como uma ferramenta disponível. No setup do Hermes, hermes setup tools mostra ferramentas de terminal instaladas automaticamente. Se o Gemini CLI estiver no PATH, o Hermes o detecta automaticamente. Caso contrário, adicione manualmente: hermes tools add gemini --command "gemini".

Por que aprender:

A integração como ferramenta é o que permite delegar tarefas multimodais ao Hermes sem precisar chamar o Gemini manualmente. O Hermes decide quando usar Gemini com base na natureza da tarefa.

Conceitos-chave:

hermes setup tools · Detecção automática via PATH · hermes tools add · Delegação automática de multimodal · Verificação de ferramenta disponível

O que é:

Com o Gemini CLI integrado, você pode pedir ao Hermes: "Use o Gemini CLI para analisar meu vídeo do YouTube [URL] e me dê um breakdown dos primeiros 30 segundos — o que aparece na tela, qual é o gancho visual e qual é a energia transmitida". O Hermes invoca o Gemini, processa o vídeo e retorna a análise estruturada.

Por que aprender:

Ver um caso de uso concreto e de alto valor demonstra por que vale a pena configurar o Gemini CLI. Análise de vídeo é algo que nenhum outro modelo no Triad faz bem — Gemini é o especialista.

Conceitos-chave:

Análise de vídeo do YouTube · Breakdown visual de conteúdo · Hermes como orquestrador · Gemini como especialista multimodal · Output estruturado

O que é:

O mesmo padrão de integração do Gemini CLI funciona para outros CLIs: GitHub CLI (gh) para criar issues e PRs via Hermes, OpenAI CLI para tarefas específicas da OpenAI, Vercel CLI para deploy, e qualquer ferramenta de terminal que o Hermes possa detectar ou configurar. O ecossistema de CLIs expande o que o Hermes pode fazer sem precisar de interfaces web.

Por que aprender:

Entender que o Gemini CLI é apenas um exemplo de um padrão extensível muda como você pensa sobre o que o Hermes pode fazer. Qualquer CLI instalado pode se tornar uma ferramenta do Hermes.

Conceitos-chave:

gh CLI para GitHub · Vercel CLI para deploy · Padrão extensível · Hermes como orquestrador de CLIs · Ecossistema local de ferramentas

🎭 Criando a skill Orpheus — Seu deep work agent

Configure o Pantheon, escreva os três system prompts do Triad e teste Orpheus com uma tarefa real.

O que é:

Skills no Hermes são configurações de comportamento especializado — uma combinação de system prompt personalizado, modelo específico, fluxo de execução e conjunto de ferramentas disponíveis. Quando você ativa uma skill, o Hermes muda sua forma de responder e executar tarefas para seguir o padrão definido na skill. Skills podem ser persistentes (sempre ativas) ou invocadas on-demand.

Por que aprender:

Entender skills é o que permite criar comportamentos especializados dentro do Hermes — como o fluxo Triad — sem precisar repetir instruções longas a cada conversa.

Conceitos-chave:

System prompt personalizado por skill · Modelo específico por skill · Invocação on-demand vs persistente · Comportamento especializado · Skills como módulos de comportamento

O que é:

O Pantheon é a interface visual do Hermes para gerenciar personas e skills. Acessível via hermes pantheon ou pelo dashboard, ele mostra todas as suas personas ativas, permite criar novas, editar system prompts, configurar modelos por persona e sincronizar com o Hermes. Cada persona no Pantheon aparece como um "deus" com nome, missão e capacidades definidas.

Por que aprender:

O Pantheon torna o gerenciamento de múltiplas personas visual e intuitivo. Sem ele, você precisaria editar arquivos de configuração diretamente — mais propenso a erros e mais difícil de visualizar.

Conceitos-chave:

hermes pantheon · Persona como entidade · Visual vs arquivo de config · Sincronização com Hermes · Nome e missão de persona

O que é:

No Pantheon, clique em "Add Persona", defina o nome como "Orpheus", a descrição como "Deep work agent — usa o fluxo Triad para tarefas estratégicas complexas" e selecione Claude Opus 4 como modelo orquestrador. A persona criada aparece no dashboard e pode ser invocada via "Use o Orpheus para..." no chat do Hermes.

Por que aprender:

Criar a persona Orpheus é o momento em que o sistema Triad se torna uma entidade nomeada e invocável dentro do Hermes. Dar um nome ajuda a comunicar a intenção — "preciso de Orpheus" sinaliza trabalho profundo.

Conceitos-chave:

Add persona no Pantheon · Nome Orpheus · Modelo orquestrador · Invocação por nome · Deep work como intenção

O que é:

O sistema Orpheus usa três system prompts distintos. Conductor (Opus): interroga com 5-10 perguntas, escreve um briefing de uma página para o Worker, valida quando Worker + Crítico convergirem — nunca executa o trabalho ele mesmo. Worker (DeepSeek): recebe o briefing e ataca de 3-5 ângulos em paralelo, passa cada draft ao Critic, revisa até receber SHIP. Critic (GPT): retorna VERDITO (SHIP/REVISE/FUNDAMENTAL FLAW), O QUE QUEBRA, O QUE FALTA e MELHOR ÂNGULO — com exemplos específicos.

Por que aprender:

Esses são os system prompts exatos que definem o comportamento do Orpheus. Entendê-los permite ajustá-los para seus casos de uso específicos.

Conceitos-chave:

Conductor nunca executa · Worker usa 3-5 ângulos · Critic retorna SHIP/REVISE/FUNDAMENTAL FLAW · Específico vs vago · Critério de aprovação rigoroso

O que é:

Após criar o Orpheus no Pantheon, sincronize com o Hermes via prompt de contexto: "Hermes, estou sincronizando a persona Orpheus com você. Orpheus é meu agente de deep work que usa o fluxo Triad: Opus como Condutor (interroga, brifa, valida), DeepSeek como Worker (executa de 3-5 ângulos, loop com Crítico), GPT-5.5 como Crítico (SHIP/REVISE/FUNDAMENTAL FLAW). Confirme que você entendeu o fluxo e me diga o que precisa antes de receber uma tarefa para Orpheus."

Por que aprender:

A sincronização via prompt confirma que o Hermes entendeu o fluxo corretamente antes de você investir em uma tarefa real. É como fazer o Hermes "ler o manual" em voz alta.

Conceitos-chave:

Prompt de sincronização · Confirmação de entendimento · Detecção de gaps antes da tarefa · Feedback do Hermes sobre o que precisa · Iteração de configuração

O que é:

O primeiro teste real do Orpheus deve ser uma tarefa de médio valor estratégico — importante o suficiente para valer a profundidade do Triad, mas não crítica ao ponto de que um resultado imperfeito seja prejudicial. Exemplo: "Orpheus, quero entender quais são os 3 principais erros que empreendedores cometem ao usar IA nos primeiros 90 dias. Me dê sua análise mais profunda." O Orpheus vai interrogar, briefar, executar e criticar.

Por que aprender:

O primeiro teste real confirma se o fluxo funciona de ponta a ponta no seu setup específico. Problemas de configuração que não aparecem em testes simples surgem em tarefas com múltiplas etapas e modelos.

Conceitos-chave:

Tarefa de médio valor · Fluxo ponta a ponta · Identificação de problemas de configuração · Interrogação como primeira etapa · Resultado convergido

O que é:

Após o primeiro uso real do Orpheus, avalie: o Condutor fez as perguntas certas? O Worker produziu ângulos genuinamente distintos? O Crítico foi específico o suficiente? Se o Condutor foi muito verboso, ajuste o system prompt para "máximo 5 perguntas, uma de cada vez". Se o Worker produziu ângulos similares, adicione "cada ângulo deve usar uma premissa fundamentalmente diferente". Iteração é esperada.

Por que aprender:

O primeiro uso raramente é perfeito. A capacidade de identificar onde ajustar e como ajustar é o que separa quem tem um sistema funcional de quem tem um sistema excelente.

Conceitos-chave:

Avaliação pós-uso · Ajuste de system prompt · Iteração esperada · Diagnóstico de qualidade · Critérios de bom output por papel

📋 soul.md na prática — Preenchendo e ativando

Do template em branco ao contexto pessoal completo — Identity, Mission, Business, Voice, Rhythm e Memory.

O que é:

O template soul.md está disponível no arquivo de referência do curso e no repositório do Hermes. Baixe o template para ~/.hermes/soul.md — esse é o local padrão que o Hermes carrega automaticamente. Abra o arquivo em qualquer editor de texto. As seções em itálico são instruções a substituir pelo seu conteúdo real.

Por que aprender:

Ter o template no local correto e editável é o pré-requisito para tudo o mais neste módulo. Um soul.md no local errado é ignorado pelo Hermes silenciosamente.

Conceitos-chave:

~/.hermes/soul.md · Carregamento automático · Editor de texto simples · Itálico = instrução a substituir · Localização correta

O que é:

Identity: preencha nome, pronomes, localização e fuso horário (ex: "São Paulo, BR · BRT / UTC-3"), seu papel profissional em uma frase e 3 adjetivos de brand voice ("direto, prático, sem jargão"). Mission: escreva seu headline goal em uma frase sem suavizações, liste 3 pilares concretos e escreva explicitamente o que está fora de escopo este ano.

Por que aprender:

Identity e Mission são as seções que mais impactam a relevância das sugestões do Hermes. Um headline goal vago ("crescer minha marca") produz sugestões genéricas. Um headline específico ("atingir 10k seguidores no LinkedIn até dezembro vendendo consultoria de IA") produz sugestões acionáveis.

Conceitos-chave:

Fuso horário explícito · 3 adjetivos de brand voice · Headline goal sem suavizações · 3 pilares concretos · Out-of-scope como proteção

O que é:

Na seção Business, preencha entidade legal (nome e estrutura), modelo de receita (SaaS / agência / cursos / consultoria), receita mensal atual e tendência percentual MoM, custos mensais detalhados (ferramentas, pessoal, infraestrutura), runway em meses se receita caísse a zero, e quais métricas o Hermes deve monitorar por canal.

Por que aprender:

Sem contexto financeiro, o Hermes recomenda o que é tecnicamente possível — não o que é viável para seu estágio. "Contrate mais 3 pessoas" é uma sugestão ruim se você tem 2 meses de runway.

Conceitos-chave:

Receita e tendência MoM · Runway como restrição real · Custo detalhado por categoria · Métricas por canal · Contexto que calibra viabilidade

O que é:

Voice: liste como quer ser respondido (curto por padrão, uma pergunta por vez, sem emojis, sem bullet points desnecessários), e como o Hermes deve escrever no seu estilo (registro, ortografia brasileira, capitalização, palavras proibidas). Hard nos: liste o que o Hermes nunca pode fazer — ex: "nunca afirmar métricas que não compartilhei", "nunca mencionar [concorrente X] pelo nome", "nunca sugerir [abordagem Y] — já testei e não funciona".

Por que aprender:

Voice e Hard nos são as seções que mais evitam retrabalho. Sem elas, você passa horas pedindo ao Hermes para mudar o formato, o tom ou retirar informações que não deveria ter incluído.

Conceitos-chave:

Formato padrão de resposta · Ortografia como identidade · Palavras proibidas · Hard no como proteção de posicionamento · Histórico de tentativas fracassadas

O que é:

Na seção Rhythm, mapeie sua semana real: quando você tem energia máxima para trabalho criativo (ex: 6h-9h), quais dias são para criação vs execução vs reuniões, quando você checa mensagens (duas vezes por dia, não em tempo real), quais dias são sem reuniões, e a curva de energia ao longo do dia (manhã: criativo; tarde: operacional; noite: revisão e leitura).

Por que aprender:

O Hermes usa o ritmo para sugerir quando executar diferentes tipos de tarefas. Uma análise profunda sugerida para as 16h vai ser ignorada. A mesma análise sugerida para amanhã às 7h tem chance real de ser feita.

Conceitos-chave:

Janela de trabalho profundo · Dias de criação vs execução · Check de mensagens em batch · Curva de energia · Rituais de recuperação

O que é:

Memory é dividida em quatro sub-seções: (1) Decisões — liste o que você tentou, por que não funcionou e o que não sugerir de novo; (2) Pessoas — nomes, papel, última interação e o que você deve/eles devem a você; (3) Ferramentas — apenas as que sobreviveram ao uso real; (4) Lições trimestrais — o que funcionou, o que não funcionou, o que surpreendeu. Seja específico — "tentei landing pages em inglês, CTR foi 40% menor que em português" é útil; "inglês não funciona" é inútil.

Por que aprender:

A seção Memory é a que cresce mais em valor com o tempo. Registrar uma decisão hoje evita que o Hermes sugira a mesma abordagem daqui a seis meses — depois que você esqueceu por que descartou.

Conceitos-chave:

Anti-sugestão por decisão documentada · Rede de pessoas como contexto · Stack real vs aspiracional · Lições trimestrais como input · Especificidade como requisito

O que é:

Com o soul.md preenchido em ~/.hermes/soul.md, reinicie o Hermes ou execute hermes reload soul. Para verificar que o arquivo foi carregado corretamente, pergunte: "Hermes, o que você sabe sobre mim? Resuma meu perfil com base no que tem no soul.md". Se a resposta incluir detalhes específicos que você preencheu, o carregamento funcionou. Se for genérica, verifique o caminho do arquivo e a sintaxe do Markdown.

Por que aprender:

A verificação final fecha o loop da configuração. Um soul.md mal posicionado ou com erro de sintaxe é silenciosamente ignorado pelo Hermes, resultando em respostas genéricas sem indicação do problema.

Conceitos-chave:

hermes reload soul · Pergunta de verificação · Resposta específica = sucesso · Resposta genérica = problema · Diagnóstico de caminho e sintaxe
