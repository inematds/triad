💡 O que é CLI e por que o Gemini CLI é poderoso

CLI significa Command Line Interface — interação via terminal sem interface gráfica. O Gemini CLI é a implementação oficial do Google, e o que o torna especialmente poderoso é que o Hermes pode invocá-lo como uma ferramenta local — transformando Gemini em capacidade multimodal sempre disponível.

O que é

CLI é uma interface de linha de comando — uma ferramenta que roda no terminal e pode ser invocada por outros programas. O Gemini CLI é a implementação oficial do Google que traz análise multimodal (vídeo, imagem, PDF) para dentro do ecossistema do Hermes.

Modelos de IA convencionais

  • •Interface gráfica obrigatória
  • •Não podem ser invocados por outros agentes
  • •Contexto isolado por sessão

Gemini CLI local

  • •Invocável pelo Hermes como ferramenta
  • •Analisa vídeo, imagem, PDF, áudio
  • •Custo zero com conta Google gratuita

🔑 Conceitos-chave

CLI vs interface gráfica

Terminal permite invocação por outros programas — interfaces gráficas não

Hermes invocando CLI

Hermes pode chamar gemini como ferramenta de terminal automaticamente

Multimodal local

Vídeo, imagem, PDF e áudio processados localmente via terminal

Custo zero

Tier gratuito do Gemini é generoso para uso pessoal e inclui Gemini 2.0

📦 Instalando o Gemini CLI

A instalação é feita via npm com uma única linha de comando. A instalação global via npm é o caminho mais simples e garante atualizações automáticas quando o Google lança novas versões.

O que é

O processo de instalar o pacote @google/gemini-cli globalmente via npm para que o comando gemini fique disponível em qualquer diretório e detectável pelo Hermes.

🛠️ Instalação e verificação

Instalar globalmente:

npm install -g @google/gemini-cli

Verificar instalação:

gemini --version

Se retornar o número da versão, a instalação está correta. Se retornar "command not found", tente reiniciar o terminal ou verificar o PATH.

🔑 Conceitos-chave

npm install -g

Instalação global que torna o comando disponível em qualquer diretório

gemini --version

Verificação que confirma instalação bem-sucedida

Atualização automática

npm update -g @google/gemini-cli atualiza para a versão mais recente

Instalação global vs local

Global (-g) é necessária para o Hermes detectar o CLI no PATH do sistema

🔐 Autenticando com sua conta Google

A autenticação via conta Google elimina a necessidade de chave de API — o que significa custo zero para começar. O tier gratuito inclui acesso generoso ao Gemini 2.0 para uso pessoal.

O que é

O fluxo OAuth via gemini auth login que autentica o CLI com sua conta Google pessoal — sem precisar de API key, Google Cloud ou cartão de crédito para o tier gratuito.

🔑 Processo de autenticação

1.Execute gemini auth login no terminal
2.O CLI abre um link de autenticação no browser automaticamente
3.Autentique com sua conta Google no browser
4.Cole o código de verificação no terminal quando solicitado
5.A autenticação é salva localmente e válida por meses

💡 Auth vs API key

A autenticação via conta Google é diferente de usar uma API key do Google Cloud. Com a conta Google pessoal, você usa o tier gratuito sem precisar de projeto no Google Cloud, cartão de crédito ou configuração adicional. Para uso comercial intensivo, use API key via Google AI Studio.

🔑 Conceitos-chave

gemini auth login

Comando que inicia o fluxo de autenticação OAuth com Google

Tier gratuito

Limite generoso para uso pessoal — suficiente para o ecossistema do Triad

Sem API key necessária

Conta Google pessoal é suficiente — sem Google Cloud ou cartão

Validade da autenticação

Token salvo localmente permanece válido por longo período

⌨️ Comandos básicos — primeiras chamadas no terminal

Com o CLI autenticado, os comandos básicos cobrem texto, arquivos e análise multimodal. Familiaridade com a sintaxe do CLI é necessária antes de integrá-lo ao Hermes — saber o que funciona diretamente facilita o debugging da integração.

O que é

A sintaxe básica do Gemini CLI para perguntas de texto, análise de arquivos com a flag -f e processamento de URLs — os comandos que o Hermes usará ao delegar tarefas multimodais.

💻 Sintaxe dos comandos

Pergunta de texto simples:

gemini "explique o que é arquitetura multi-modelo"

Análise de PDF:

gemini -f relatorio.pdf "resuma os principais pontos"

Análise de imagem:

gemini -f screenshot.png "descreva o que você vê"

Análise de vídeo (URL):

gemini -f https://youtube.com/watch?v=XYZ "analise os primeiros 30 segundos"

🔑 Conceitos-chave

gemini "prompt"

Sintaxe básica para qualquer pergunta de texto

-f para arquivo

Flag que passa um arquivo ou URL como entrada para análise

Tipos suportados

PDF, imagens (PNG/JPG), vídeos, áudio e URLs do YouTube

Saída no terminal

Resultado é texto que pode ser capturado e processado por outros comandos

🔗 Conectando o Gemini CLI ao Hermes como ferramenta

Para que o Hermes possa usar o Gemini CLI, configure-o como ferramenta disponível. Se o CLI estiver no PATH, o Hermes o detecta automaticamente — a integração pode ser zero-configuração.

O que é

O processo de registrar o Gemini CLI como ferramenta no Hermes via hermes setup tools — após isso, o Hermes delega automaticamente tarefas multimodais ao Gemini sem precisar ser instruído explicitamente.

🛠️ Configurando a integração

Verificar ferramentas detectadas automaticamente:

hermes setup tools

Se o Gemini CLI estiver no PATH, aparece na lista automaticamente.

Se não detectar automaticamente, adicionar manualmente:

hermes tools add gemini --command "gemini"

💡 Delegação automática

Com o Gemini CLI configurado como ferramenta, o Hermes decide automaticamente quando usá-lo — você não precisa especificar "use o Gemini para isso". Se a tarefa envolve análise visual ou multimodal, o Hermes invoca o Gemini por conta própria. Isso é a diferença entre ter uma ferramenta disponível e ter uma ferramenta integrada.

🔑 Conceitos-chave

hermes setup tools

Wizard que detecta e configura ferramentas de terminal disponíveis

Detecção automática

CLIs no PATH do sistema são detectados sem configuração manual

hermes tools add

Comando para adicionar manualmente ferramentas não detectadas

Delegação automática

Hermes decide quando usar Gemini com base na natureza da tarefa

🎬 Caso de uso: analisar vídeo/imagem via Hermes

Com o Gemini CLI integrado, análise de vídeo e imagem se torna uma capacidade natural do Hermes. Análise de vídeo é algo que nenhum outro modelo no Triad faz bem — Gemini é o especialista multimodal do ecossistema.

O que é

Um fluxo onde o Hermes recebe um pedido de análise de vídeo ou imagem, invoca automaticamente o Gemini CLI com a URL ou arquivo, e entrega o breakdown estruturado — sem que você precise interagir com o CLI diretamente.

🎯 Caso de uso concreto

Prompt para o Hermes:

"Use o Gemini CLI para analisar meu vídeo do YouTube [URL] e me dê um breakdown dos primeiros 30 segundos — o que aparece na tela, qual é o gancho visual e qual é a energia transmitida"

O que acontece nos bastidores:

1.Hermes recebe o pedido e identifica que precisa de análise de vídeo
2.Invoca o Gemini CLI automaticamente com a URL do vídeo
3.Gemini analisa o vídeo e retorna dados visuais estruturados
4.Hermes formata e entrega o breakdown para você

🔑 Conceitos-chave

Análise de vídeo do YouTube

Gemini processa URLs diretamente — sem download necessário

Breakdown visual

Análise estruturada de elementos visuais, narrativa e energia do conteúdo

Hermes como orquestrador

Hermes coordena a chamada ao Gemini e processa o resultado

Especialista multimodal

Gemini é o único modelo do ecossistema com capacidade nativa de vídeo

🧩 Expandindo para GitHub CLI, OpenAI CLI e outros

O Gemini CLI é um exemplo de um padrão extensível. Qualquer CLI instalado pode se tornar uma ferramenta do Hermes — o ecossistema de CLIs transforma o Hermes em um orquestrador de capacidades locais ilimitadas.

O que é

O padrão de integração que permite adicionar qualquer CLI ao ecossistema do Hermes — GitHub CLI, Vercel CLI, AWS CLI e qualquer outro que esteja no PATH e retorne texto podem ser registrados e invocados automaticamente.

🔌 CLIs que seguem o mesmo padrão

GitHub CLI (gh)

Cria issues, PRs e releases via Hermes sem abrir o browser

Vercel CLI

Deploy de projetos web diretamente pelo Hermes

OpenAI CLI

Acesso a modelos e fine-tuning da OpenAI via terminal

AWS CLI / gcloud

Infraestrutura cloud gerenciada pelo Hermes

Stripe CLI

Operações de pagamento e testes de webhook

Qualquer CLI local

Se está no PATH e retorna texto, o Hermes pode usá-lo

🔑 Conceitos-chave

gh CLI para GitHub

Hermes pode criar issues e PRs sem sair do fluxo principal

Padrão extensível

O mesmo processo de configuração funciona para qualquer CLI

Hermes como orquestrador

Camada de inteligência que coordena múltiplas ferramentas especializadas

Ecossistema local

CLIs instalados localmente ampliam capacidades sem dependência de interfaces web
