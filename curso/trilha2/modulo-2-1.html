💻 Pré-requisitos — o que você precisa antes de começar

Antes de instalar o Hermes, confirme que você tem o ambiente correto. Verificar pré-requisitos antes de instalar evita erros frustrantes no meio do processo — a maioria dos problemas de instalação vem de versões incorretas de dependências ou falta de permissões adequadas.

O que é

Um conjunto mínimo de ferramentas e permissões que o ambiente precisa ter antes de instalar o Hermes — como Node.js 18+, terminal funcional e conta Anthropic ativa.

📋 Pré-requisitos obrigatórios

  • •Sistema operacional: macOS, Linux ou Windows com WSL (Windows Subsystem for Linux)
  • •Node.js 18+: verifique com node --version
  • •Conta Anthropic: para autenticar o Claude como modelo orquestrador
  • •Terminal: com permissões de usuário (não precisa de sudo para tudo)
  • •Editor de código: VS Code recomendado, mas qualquer editor funciona

💡 Dica Prática

Não é necessário ser desenvolvedor para usar o Hermes, mas familiaridade com o terminal é essencial. Se você consegue navegar entre diretórios e rodar comandos simples, está pronto. O Hermes abstrai toda a complexidade técnica — a instalação é o momento com mais fricção do processo inteiro.

🔑 Conceitos-chave

Node.js como runtime

O motor de execução JavaScript que o Hermes usa internamente

Terminal/shell básico

Navegação, execução de comandos e variáveis de ambiente

WSL para Windows

Camada de compatibilidade Linux que permite rodar Hermes no Windows

Permissões de usuário

Instalação local (~/...) não requer privilégios de administrador

📦 Instalação do Hermes Agent — passo a passo

O Hermes Agent é instalado via terminal. Seguir a sequência correta de instalação evita estados inconsistentes que são difíceis de diagnosticar depois. Pular o wizard de setup inicial é a causa número um de configurações quebradas.

O que é

O processo de instalação global do Hermes via npm e execução do wizard interativo que configura chaves de API, modelos padrão e a estrutura de diretórios necessária.

1

Instalar via npm

npm install -g hermes-agent

A flag -g instala globalmente, tornando o comando hermes disponível em qualquer diretório do terminal.

2

Rodar o wizard de setup

hermes setup

O wizard interativo guia pela configuração de chaves de API, preferências de modelo e criação da estrutura de diretórios. Não pule nenhuma etapa.

3

Verificar a instalação

hermes --version

Deve retornar o número da versão instalada. Se retornar "command not found", a instalação não foi concluída corretamente.

🔑 Conceitos-chave

npm install global

Instalação acessível de qualquer diretório no terminal

hermes setup wizard

Configuração interativa que não deve ser pulada

Variáveis de ambiente

ANTHROPIC_API_KEY e outras chaves configuradas no .env ou shell

Estado de configuração

Sequência de setup que deve ser seguida para evitar inconsistências

📁 Configuração inicial — diretórios, permissões, variáveis

Após a instalação base, o Hermes cria uma estrutura de diretórios em ~/.hermes/. Conhecer essa estrutura permite fazer backup do que importa, entender onde fica cada configuração e diagnosticar problemas com mais rapidez quando algo falha.

O que é

A árvore de diretórios criada em ~/.hermes/ após a instalação — incluindo soul.md, config.json, skills/ e memory/ — e as variáveis de ambiente necessárias para autenticar o Hermes.

🗂️ Estrutura de diretórios ~/.hermes/

soul.md

Contexto pessoal — quem você é, seus objetivos, negócio e preferências

skills/

Personas e skills configuradas — incluindo Orpheus e outras

memory/

Histórico de conversas e memória persistente entre sessões

config.json

Configurações gerais — modelos, chaves de API, preferências de execução

🔐 Variáveis de ambiente necessárias

ANTHROPIC_API_KEY

Obrigatória — autentica o Claude como modelo padrão do Hermes

OPENROUTER_API_KEY

Opcional mas recomendada — habilita acesso a DeepSeek, GPT e outros modelos

🔑 Conceitos-chave

~/.hermes/ como base

Diretório raiz de toda a configuração pessoal do Hermes

ANTHROPIC_API_KEY

Chave obrigatória para o modelo orquestrador funcionar

Backup de configuração

Copie ~/.hermes/ regularmente — é onde seu trabalho de configuração vive

config.json

Arquivo central de configuração — editável diretamente quando necessário

🧪 Primeiro teste — confirmar que Hermes está funcionando

Após a instalação e configuração, o primeiro teste é uma conversa simples para confirmar que o sistema responde corretamente. Testar o básico antes de avançar é a prática que separa instalações que funcionam de instalações que parecem funcionar mas falham em casos importantes.

O que é

Um smoke test em dois comandos que verifica se o Hermes lê o soul.md e consegue executar ferramentas de terminal — os dois pilares da funcionalidade básica do sistema.

✓ Teste 1 — Memória e soul.md

hermes "O que você sabe sobre mim até agora?"

Se o Hermes mencionar dados que você preencheu no soul.md, o carregamento de contexto está funcionando.

✓ Teste 2 — Execução de ferramenta

hermes "Crie um arquivo ~/Desktop/hermes-test.txt com o texto 'Instalação OK'"

Se o arquivo for criado, o Hermes está executando ferramentas de terminal corretamente.

💡 Por que esses dois testes específicos

O Teste 1 valida que o soul.md está sendo lido — o problema mais comum em instalações parciais. O Teste 2 valida que o Hermes tem permissão para executar comandos de terminal — necessário para a maioria dos casos de uso avançados. Juntos, eles cobrem os dois pilares da funcionalidade do Hermes: contexto e execução.

🔑 Conceitos-chave

Smoke test

Teste mínimo que confirma que o sistema básico funciona

Verificação de soul.md

Confirma que o contexto pessoal está sendo carregado

Execução de terminal

Hermes precisa de permissão para criar/modificar arquivos

Iteração de diagnóstico

Isolar o problema em componentes para encontrar a causa raiz

📊 O dashboard do Hermes — visão geral da interface

O Hermes tem uma interface de dashboard que centraliza todas as informações do sistema em um lugar visual. Conhecer o dashboard antes de precisar dele reduz a fricção quando algo vai errado — você sabe onde olhar sem precisar procurar.

O que é

A interface visual do Hermes acessada com hermes dashboard — um painel que centraliza histórico de conversas, memória ativa, personas, uso de tokens e jobs agendados.

🖥️ O que você encontra no dashboard

Histórico e memória

  • •Histórico completo de conversas
  • •Memória ativa e contexto carregado
  • •Decisões registradas do soul.md

Controle operacional

  • •Skills e personas ativas (Pantheon)
  • •Uso e custo de tokens acumulado
  • •Status de jobs agendados em background

🔑 Conceitos-chave

hermes dashboard

Comando para abrir a interface visual no browser

Pantheon visual

Interface gráfica para gerenciar personas e skills

Status de jobs

Monitoramento de tarefas executadas em background

Custo acumulado

Rastreamento de uso de tokens e custo por período

6

🔗 Conectando Claude Code ao Hermes — integração

Claude Code e Hermes são ferramentas complementares que funcionam melhor integradas. A integração entre os dois é onde o sistema atinge seu potencial máximo — Hermes orquestra a vida toda, incluindo o trabalho de código que Claude Code executa.

O que é

A configuração via hermes setup integrations que conecta Claude Code ao Hermes — permitindo que o Hermes delegue tarefas de código automaticamente e compartilhe contexto entre as duas ferramentas.

⚡ Configurando a integração

hermes setup integrations

O wizard de integrações mostra as ferramentas detectadas no sistema e permite configurar como o Hermes as utiliza.

O que a integração permite

  • •Hermes delega tarefas de código ao Claude Code automaticamente
  • •Dashboard unificado mostra uso de ambas as ferramentas
  • •Contexto compartilhado — Hermes sabe o que Claude Code fez

🔑 Conceitos-chave

hermes setup integrations

Wizard para conectar ferramentas externas ao Hermes

Delegação de código

Hermes decide quando usar Claude Code para tarefas técnicas

Dashboard unificado

Visão consolidada de uso e custo de todas as ferramentas

Fluxo de orquestração

Hermes como camada de coordenação entre ferramentas especializadas

✅ Verificação de saúde — checklist pós-instalação

Antes de avançar para qualquer configuração avançada, execute o checklist completo de verificação de saúde. Um sistema parcialmente configurado pode parecer funcionar mas falhar em casos importantes — o checklist garante que você tem uma base sólida.

O que é

Uma lista de sete verificações que cobrem os componentes essenciais do Hermes — resposta básica, carregamento do soul.md, chave ativa, execução de terminal, dashboard, memória persistente e skills disponíveis.

🩺 Checklist de verificação de saúde

✓
Hermes responde

Execute hermes "olá" e confirme resposta

✓
soul.md é carregado

Hermes menciona dados do seu perfil sem você os fornecer

✓
Chave API está ativa

Nenhum erro de autenticação nas primeiras chamadas

✓
Terminal funciona como ferramenta

Hermes consegue criar arquivos e executar comandos

✓
Dashboard abre

Execute hermes dashboard e confirme que abre no browser

✓
Histórico é salvo entre sessões

Feche e reabra o terminal — Hermes ainda lembra a conversa anterior

✓
Skills podem ser carregadas

Execute hermes skills list sem erro

🔑 Conceitos-chave

Verificação sistemática

Testar cada componente antes de considerar a instalação concluída

Base sólida antes de avançar

Configurações avançadas falham se a base estiver incompleta

Diagnóstico de instalação

Cada item do checklist mapeia para um componente específico do sistema

Problemas latentes

Erros não detectados agora causam falhas silenciosas em uso real
